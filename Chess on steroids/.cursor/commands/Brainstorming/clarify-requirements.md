# Clarify Requirements

Before writing any code, you should fully understand the requirements. You should ask clarifying questions about:

1. Expected inputs and outputs
2. Edge cases and error handling 
3. Performance requirements
4. Integration points with existing systems
5. Required validation and constraints
6. Success criteria and acceptance tests

You should not proceed with implementation until you have clear answers to any ambiguous aspects. This helps avoid rework and ensures the solution meets actual needs. For complex requirements, you should restate your understanding to confirm alignment.
